📚 アルゴリズムの解説

🚀 クイックソート (Quick Sort)

概要: 分割統治法を用いたソートアルゴリズム。ピボット（基準値）を選び、それより小さい要素と大きい要素に分割を繰り返す。

時間計算量:

  • 最良: O(n log n)
  • 平均: O(n log n)
  • 最悪: O(n²) ※ピボット選択が偏った場合

空間計算量: O(log n) (再帰のスタック)

安定性: 不安定

特徴: 平均的に非常に高速で実用的。in-placeなアルゴリズム。

🔀 マージソート (Merge Sort)

概要: 配列を半分ずつ分割し、それぞれをソートした後にマージ（併合）する分割統治法。

時間計算量:

  • 最良: O(n log n)
  • 平均: O(n log n)
  • 最悪: O(n log n) ※常に一定

空間計算量: O(n) (追加の配列が必要)

安定性: 安定

特徴: 時間計算量が常にO(n log n)で安定。外部ソートに適している。

🔵 バブルソート (Bubble Sort)

概要: 隣接する要素を比較し、順序が逆なら交換を繰り返す。大きい要素が泡のように浮き上がる。

時間計算量:

  • 最良: O(n) ※既にソート済みの場合
  • 平均: O(n²)
  • 最悪: O(n²)

空間計算量: O(1)

安定性: 安定

特徴: 最もシンプルだが効率は悪い。教育用途に適している。

🔍 選択ソート (Selection Sort)

概要: 未整列部分から最小値を選択し、先頭に配置することを繰り返す。

時間計算量:

  • 最良: O(n²)
  • 平均: O(n²)
  • 最悪: O(n²)

空間計算量: O(1)

安定性: 不安定

特徴: 交換回数が少ない。シンプルだが効率は悪い。

📥 挿入ソート (Insertion Sort)

概要: 整列済み部分に要素を適切な位置に挿入していく。トランプの手札整理に似ている。

時間計算量:

  • 最良: O(n) ※既にソート済みの場合
  • 平均: O(n²)
  • 最悪: O(n²)

空間計算量: O(1)

安定性: 安定

特徴: 小規模データや部分的にソート済みのデータに効率的。
